### 我的理解

线性回归：
线性回归是一种运用线性模型拟合，达到预测输出的方式，输出与各输入呈线性关系。

数据迭代：
要弄懂这个概念，就必须知道可迭代对象和迭代器的区别，例如列表就是一个可迭代对象，可以顺着不同下标对列表进行迭代输出不同的元素；迭代器则是对可迭代对象进行迭代操作的东西，一般用iter()创建，可以利用next()取出不同的元素。迭代器本身也是可迭代对象，对迭代器调用 `iter()`，通常返回它自己，也就是```
``` python
lst = [1, 2, 3] 
iterator = iter(lst) 
iter(iterator) is iterator # True
```
因此，在读取数据集时，我们可以利用一下代码取出批量的数据。
``` python
def data_iter(batch_size, features, labels):
    num_examples = len(features)
    indices = list(range(num_examples))
    # 这些样本是随机读取的，没有特定的顺序
    random.shuffle(indices)
    for i in range(0, num_examples, batch_size):
        batch_indices = torch.tensor(
            indices[i: min(i + batch_size, num_examples)])
        yield features[batch_indices], labels[batch_indices]

batch_size = 10
for X, y in data_iter(batch_size, features, labels):
    print(X, '\n', y)
    break
```
代码中的 `for` 循环会对数据进行迭代。`for ... in ...` 会先对后面的可迭代对象调用 `iter()`，得到对应的迭代器，再不断调用 `next()` 获取元素。例如，列表是可迭代对象，`for` 循环会自动为其创建迭代器。
`data_iter` 中使用了 `yield`，因此它是生成器函数。调用 `data_iter(batch_size, features, labels)` 后会得到生成器对象，而生成器对象同时也是迭代器和可迭代对象。对迭代器调用 `iter()` 通常返回它本身，因此该生成器对象可以直接用于：
``` python
for X, y in data_iter(batch_size, features, labels):
```

自动微分与计算图：
在一些计算过程（最典型的如前向传播）中==（即梯度模式开启，运算涉及 `requires_grad=True` 的张量时）==，pytorch的动态计算图会记录下参数的依赖关系和计算路径，在反向传播（backward）的时候梯度计算则会沿着原有的路径。因此为了防止占用额外空间和避免得出错误的计算结果，需要用到```torch.no_grad()```，也就是不将此计算过程记录进计算图，常见的使用区域有train_loss的计算、反向传播完成后的参数更新等等，前者能避免额外生成无用的计算图；后者能防止更新参数的优化过程随新一次前向传播被记录到计算图中，同时还可以防止同一个参数对自身进行-=操作，旧值变成新值，难以安全管理参数的依赖关系。
==补充：进一步理解了自动微分系统与 batch 并行计算。对于一个 batch，模型会利用张量运算并行完成多个样本的前向计算，得到各样本对应的 loss，再通过 `mean()` 求出 batch 的平均损失，最后执行反向传播。
平均后的损失虽然只是一个标量，但仍然可以向前追溯并求导，因为计算图在前向传播时已经记录了参数、张量及各个算子之间的依赖关系。自动微分系统并不是根据最终数值重新识别某部分结果来自哪个参数，而是沿着已经建立的计算图，根据加法、乘法、均值等算子的局部求导规则，使用链式法则逐层传播梯度。
其中，`mean()` 本身也是计算图中的一个算子。若 batch 中有 (B) 个样本，则平均损失对每个样本损失的导数均为 (1/B)。因此，反向传播后得到的参数梯度相当于该 batch 中各样本梯度的平均值。==

batch:
要理解batch，就需要理解平均梯度。实际训练中有很多的样本对应不同的loss。优化过程就例如在loss的山地中找到谷底。不同样本的loss的梯度可能指向不同的方向，例如y1说往东走，y2说往南走，因此我们可以取他们意见的平均值（东南方向），作为下山的路线，这就是平均梯度，即整个 batch 会一起完成前向传播，损失求平均后再执行一次反向传播，作为当前 batch 对总体梯度的估计，并用于决定本次参数更新的方向和幅度。每一次我们听取多少个样本的意见，这批样本的数量就是batchsize。若是batchsize很小，例如为1，我们就要顺着听每个人的指引一步步走，若batchsize很大，我们就可以综合更多人的意见，因此当前 batch 的梯度通常更加稳定，更接近整个数据集的平均梯度，但不代表一定能够更快收敛或取得更好的泛化性能。这就产生需要权衡的问题：若是batchsize很大，每次求平均梯度，需要同时计算很多样本的梯度，通过矢量化gpu可以并行计算这些梯度，但是显存占用更大，并行计算的能力有限==补充：方差更小、更稳定，但过大的batch可能影响泛化==；若是batchsize很小，每次求平均梯度很快，但更新次数为
``` python
math.ceil(num_examples / batch_size)
```
每步的效率较低，更新次数较多==梯度噪声更大，但噪声又可能更利于优化和泛化==。因此需要取合适的batchsize，才能更有利于训练。
### 数据集加载
还学习了pytorch官方的数据加载方式
``` python
dataset = data.TensorDataset(*data_arrays)
data_iter = data.DataLoader(dataset, batch_size, shuffle=is_train)
```
其中，`*data_arrays` 会将 `(features, labels)` 元组解包，相当于向 `TensorDataset` 分别传入特征和标签。`TensorDataset` 负责按样本索引绑定特征与标签，`DataLoader` 则负责随机打乱（可选）、划分 batch，生成一个可迭代对象（即data_iter）,并在迭代时输出 `(X, y)`。
