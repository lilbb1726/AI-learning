## 栈的定义

栈（Stack）是一种**后进先出（LIFO, Last In First Out）**的数据结构。

栈只允许在**栈顶（Top）**进行插入和删除操作。

```
      Top
       │
    ┌─────┐
    │  5  │ ← 最后进入，最先出去
    ├─────┤
    │  3  │
    ├─────┤
    │  1  │ ← 最先进入，最后出去
    └─────┘
```

---

## 栈的基本操作

|操作|作用|Python（List）|时间复杂度|
|---|---|---|---|
|入栈（Push）|在栈顶加入元素|`append(x)`|O(1)|
|出栈（Pop）|删除并返回栈顶元素|`pop()`|O(1)|
|查看栈顶|返回栈顶元素但不删除|`stack[-1]`|O(1)|
|判断是否为空|判断栈是否为空|`not stack`|O(1)|
|获取栈大小|返回元素个数|`len(stack)`|O(1)|

## Python 中使用 List 实现栈

### 创建栈

```
stack = []
```

---

### 入栈

作用：向栈顶压入一个元素。

```
stack.append(10)
```

是否修改原对象：

> 是

返回值：

> `None`

时间复杂度：

> O(1)

---

### 出栈

作用：删除并返回栈顶元素。

```
top = stack.pop()
```

是否修改原对象：

> 是

返回值：

> 栈顶元素

时间复杂度：

> O(1)

示例：

```
stack = [1, 2, 3]

top = stack.pop()

print(top)     # 3
print(stack)   # [1, 2]
```

> ⚠️ 注意
> 
> `pop()` 会**修改栈本身**。
> 
> 如果需要多次使用栈顶元素，应先保存：
> 
> ```
> top = stack.pop()
> ```
> 
> 不要多次调用 `stack.pop()`。

---

### 查看栈顶

作用：查看栈顶元素，但不删除。

```
top = stack[-1]
```

是否修改原对象：

> 否

返回值：

> 栈顶元素

时间复杂度：

> O(1)

> ⚠️ 注意
> 
> 空栈访问 `stack[-1]` 会抛出异常，应先判断：
> 
> ```
> if stack:
>     top = stack[-1]
> ```

---

### 判断栈是否为空

推荐写法：

```
if not stack:
```

不推荐：

```
if stack == None:
```

原因：

- `[]` 表示空列表
- `None` 表示空对象

两者不是同一个概念。

## 栈的典型应用

### 1. 括号匹配

基本思想：

- 左括号入栈
- 遇到右括号
    - 栈为空 → 不合法
    - 弹出栈顶
    - 判断是否匹配
- 最后栈必须为空

流程：

```
左括号
   │
   ▼
 入栈

右括号
   │
   ▼
栈为空？
 │      │
是      否
 │       │
False   Pop
          │
          ▼
     是否匹配？
      │     │
     否     是
      │      │
   False   继续

最后：

栈为空 → True

栈非空 → False
```

时间复杂度：

> O(n)

空间复杂度：

> O(n)

---

### 2. 十进制转换为二进制

基本思想：

不断进行

```
除以 2
```

保存余数。

由于余数产生顺序与二进制表示顺序相反，因此利用栈进行逆序输出。

例如：

```
10

10 ÷ 2 = 5 余0

5 ÷ 2 = 2 余1

2 ÷ 2 = 1 余0

1 ÷ 2 = 0 余1
```

余数入栈：

```
0
1
0
1
```

依次出栈：

```
1010
```

时间复杂度：

> O(log n)

空间复杂度：

> O(log n)

## List 实现栈的底层思想

Python 没有内置 `Stack` 类。

实际开发中通常使用 `list` 模拟栈：

```
stack = []

stack.append(x)

stack.pop()
```

原因：

- `append()` 在尾部插入，平均 O(1)
- `pop()` 删除尾部元素，O(1)

因此非常适合作为栈。

---

## 易错点

### `pop()` 有副作用

```
stack.pop()
```

不仅返回元素，

还会删除该元素。

正确：

```
top = stack.pop()
```

错误：

```
if stack.pop() == "(":
    ...

if stack.pop() == "[":
    ...
```

第二次 `pop()` 会继续删除元素。

---

### 空栈不能直接 Pop

错误：

```
top = stack.pop()
```

正确：

```
if not stack:
    return False

top = stack.pop()
```

---

### 判断括号是否合法

不能比较：

```
ch == top
```

因为：

```
(
)
```

本身就不同。

应比较：

```
top == "("

对应

ch == ")"
```
